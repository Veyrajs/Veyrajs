# `src/Node.svelte` ŌĆö The generic node component

> Wraps an engine node declaratively. The heart of the adapter; every wrapper delegates here.

## Purpose

A single generic component that, given a `nodeClass` + the prop `keys` to mirror, drives one
engine node: create it, attach it, mirror props, re-emit events, expose it, and remove it.
It is the Svelte counterpart of Vue's `defineNodeComponent` and React's `createNodeComponent`
ŌĆö except in Svelte a "factory" must be a real component, so the nine wrappers pass their
config in as props instead of being generated by a function.

## Props

- `nodeClass` ŌĆö the engine node constructor (e.g. `Rect`).
- `keys` ŌĆö the prop names to mirror onto the node (transform + style + per-shape extras).
- `props` ŌĆö the bag of engine props + `on*` callbacks, forwarded from the wrapper's rest.
- `isContainer` ŌĆö whether to provide a child context and render `children`.
- `node` ŌĆö **bindable**; set to the created instance (the escape hatch).
- `children` ŌĆö the container's slot.

## What it does

1. **Create once.** `const instance = untrack(() => new nodeClass(buildConfig(props, keys)))`.
   `untrack` reads the *current* prop values without subscribing, so the node is built once and
   never re-created when props change ŌĆö the prop mirror handles updates. (Without `untrack`,
   Svelte 5 warns `state_referenced_locally`.)
2. **Hand down the parent.** A container calls `setNodeContext` with a getter that returns
   *itself* as `parent`, extending the cascade to its children.
3. **Attach.** An `$effect` reads `context.parent`; when it becomes non-null it calls
   `parent.add(instance)`. Reading the getter inside the effect is what makes this re-run once
   the stage mounts.
4. **Mirror props.** An `$effect` re-applies every defined prop in `keys`. The engine's guarded
   setters (`if (v !== this._x)`) make re-application loop-safe.
5. **Re-emit events.** An `$effect` subscribes one handler per engine event; each reads
   `props[`on${type}`]` **lazily**, so Svelte's reactive `props` always supply the latest
   callback ŌĆö no need to re-subscribe when the callback identity changes.
6. **Clean up.** `onMount` returns `() => instance.remove()`, and the event effect's teardown
   `off`s the handlers.

Leaf shapes render nothing; containers render `{@render children?.()}`.

## Conventions & gotchas

- **`let`, not `const`, for `$props()`** ŌĆö Svelte requires it, even though the variables look
  write-once. (This is why `.svelte` files are excluded from Biome, whose `useConst` rule
  doesn't understand runes.)
- **Props are one-way.** No node ŌåÆ prop write-back; sync from events.
- **`untrack` marks deliberate one-time reads.** It is the Svelte analogue of React's
  "create in `useRef`, ignore later props" ŌĆö intent made explicit to the compiler.
- **The cast at construction** (`as never`) bridges the loose prop bag to the node's typed
  config ŌĆö the one type-erasure point, mirroring the serializer's registry.

## Relationships

- **Uses:** the engine node classes + event types; Svelte (`$props`, `$bindable`, `$effect`,
  `onMount`, `untrack`); [context.ts](./context.md).
- **Used by:** every wrapper in [components.md](./components.md). Exported as `ACNode` so apps
  can wrap **custom** node types (e.g. annotation primitives) the same way.
